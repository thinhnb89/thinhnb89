Welcome to the Canada Day edition of the less-than-weekly-now collection of Selenium / Automation links that is the Smattering of Selenium.

*   [Selenium Advanced User Interactions](http://www.theautomatedtester.co.uk/blog/2011/selenium-advanced-user-interactions.html) illustrates how this new API (which is part of WebDriver) can use used to do things like drag-and-drop on Canvas elements.
*   [Things you (probably) didn’t know about xargs](http://offbytwo.com/2011/06/26/things-you-didnt-know-about-xargs.html) is one of those things to keep tucked away in your toolbox. Especially if you find yourself having to shell out in your scripts.
*   [Sapphire](http://rubygems.org/gems/sapphire) is yet another Ruby framework. Yet again aiming at _non-technical resources_. Though I am sure he meant ‘people’.
*   Se-IDE 1.0.12 has been available on the website for a bit and is being pushed out to everyone this weekend. Samit has a [summary of the changes](http://blog.reallysimplethoughts.com/2011/06/23/selenium-ide-v1-0-12-is-now-here/)
*   [Automated Testing of RichFaces Components with Selenium](http://thoughtforge.net/2011/06/21/automated-testing-of-richfaces-components-with-selenium/) is a nice Maven example including how to deal with complex AJAX calls
*   [The Tests are Broken, Now What?](http://www.bryancook.net/2011/05/tests-are-broken-now-what.html) explains a method in the C# world of dealing with flaky scripts
*   [A Python Decorator to Mark Nose Tests as Work in Progress](http://www.natpryce.com/articles/000788.html)
*   [Advice on starting browser-based automation](http://testerthoughts.com/2011/06/14/advice-on-starting-browser-based-automation/) is a pretty decent list — even if it does need to breaks between points.
*   [JavaScript alerts in Selenium 2, webdriver and capybara”](http://agilesoftwaretesting.com/?p=230)\> looks like it was good before, and better with the help of some feedback
*   [Headless Selenium WebDriver Tests using PyVirtualDisplay](http://coreygoldberg.blogspot.com/2011/06/python-headless-selenium-webdriver.html)
*   [Python – Selenium WebDriver – Capture Screenshot](http://coreygoldberg.blogspot.com/2011/06/python-selenium-webdriver-capture.html)
*   [Inspecting XPath Expressions and CSS Selectors using Firefox/Chrome](http://blog.browsermob.com/2011/06/inspecting-xpath-expressions-and-css-selectors-using-firefoxchrome) via Firebug or the console
*   [Selenium IDE Plug-in for InfoStretch Framework](http://blog.infostretch.com/?p=1106) includes a formatter for their tool as well as a bunch of other extensions.
*   The [ClasspathSuite](http://johanneslink.net/projects/cpsuite.jsp) modifies how JUnit 4 finds what scripts it will run.
*   [Selenium-Specs](https://github.com/jesseeichar/Selenium-Specs) is an Se-IDE formatter for Scala / Spec2
*   [Acceptance testing a CherryPy application with Robot Framework](http://www.defuze.org/archives/266-acceptance-testing-a-cherrypy-application-with-robot-framework.html) (and Selenium)
*   I’m starting to think that Selenium needs a curated Recipe site; [The Makings of a Great Python Cookbook Recipe](http://www.protocolostomy.com/2010/12/20/the-makings-of-a-great-python-cookbook-recipe/) is an example of what I think we need
*   [Selenium 2 Chrome Switches via ChromeDriver](http://twist4all.wordpress.com/2011/06/13/chrome-switches/) for all your toggle-ing needs
*   [Test Maintenance; Or, The Third Era of Programmer Testing](http://www.exampler.com/blog/2011/06/17/test-maintenance-or-the-third-era-of-programmer-testing/) postulates that script maintenance is on the verge of an evolution
